# Implementation Plan: Smooth Camera Controls and Demo Narration

**Branch**: `004-camera-smooth-demos` | **Date**: 2026-03-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-camera-smooth-demos/spec.md`

## Summary

Add smooth camera interpolation, 6 body-relative camera modes (lookAt, follow, orbit, chase, frameBodies, shake), narration labels, and scripting helpers. Enhance all 42 demo scripts (21 F# + 21 Python) with cinematic camera sequences and narration. Extends the ViewCommand proto with 9 new message types; all interpolation runs viewer-side using frame delta time and smoothstep easing. No server changes required.

**Language/Version**: F# on .NET 10.0 (viewer, client, scripting), C# on .NET 10.0 (contracts), Python 3.10+ (demo scripts)
**Primary Dependencies**: Stride.CommunityToolkit 1.0.0-preview.62 (viewer), Grpc.AspNetCore.Server 2.x, Google.Protobuf 3.x, Grpc.Tools 2.x, grpcio (Python)

**Testing**: xUnit 2.x, Aspire.Hosting.Testing 10.x
**Target Platform**: Linux with GPU (Stride3D viewer)
**Project Type**: Multi-service desktop application (Aspire-orchestrated)
**Performance Goals**: 60 FPS during camera transitions, no frame drops
**Constraints**: Viewer-side interpolation only (fire-and-forget from scripts), backward compatible with existing instant camera commands
**Scale/Scope**: 9 new proto messages, ~15 new F# functions, ~15 new Python functions, 42 demo scripts enhanced, 1 new demo
